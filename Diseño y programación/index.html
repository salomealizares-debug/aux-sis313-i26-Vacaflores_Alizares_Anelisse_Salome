PRACTICA N° 2 - GIT,GITHUB - HTML
  1. ¿Qué es una rama (branch) en Git y para qué sirve?

    2. Una rama en Git es una copia del proyecto donde se pueden hacer cambios sin afectar la versión principal. Sirve para trabajar en nuevas funcionalidades, corregir errores o probar cosas sin arruinar el código principal.

  2. ¿Cómo se crea y se cambia entre ramas en Git?

    3. Para crear una rama se usa: git branch nombre_rama Para cambiar a esa rama: git checkout nombre_rama También se puede hacer en un solo paso: git checkout -b nombre_rama

  3. ¿Cuál es la diferencia entre la rama main y otras ramas?

    4. La rama main es la principal del proyecto, donde está la versión estable. Las otras ramas se usan para trabajar en cambios o nuevas funciones sin afectar la rama principal hasta que todo esté listo

  4. ¿Cómo se combinan ramas en Git (merge)?

    5. Primero se cambia a la rama donde quieres unir los cambios (por ejemplo main): git checkout main Luego se hace el merge: git merge nombre_rama Esto une los cambios de la otra rama con la actual.
